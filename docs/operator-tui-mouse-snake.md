# Operator TUI Mouse Snake (Windows Terminal + WSL2)

## Purpose

This guide explains how to use mouse-driven Snake navigation in `ananta tui`, how artifact intent and AI-Snake fast-targeting work, and how to validate the homepage cast flow.

## Environment checklist (Windows + WSL2)

1. Use **Windows Terminal** with a WSL2 Ubuntu shell.
2. Ensure `TERM=xterm-256color` in the shell.
3. Start TUI in the project root:

```bash
ananta tui
```

4. In the TUI status line, verify:
   - `mouse_support=enabled`
   - `term=xterm-256color` (or similar xterm-compatible value)

## Controls

- `Ctrl+S` toggle Snake mode.
- `o` toggle mouse-follow mode.
- `u` toggle Tutorial/AI-Snake mode.
- `m` message mode (ask follow-up questions).
- `e` open selected artifact/file in inline read-only viewer.
- `:mouse on|off|toggle` command-mode fallback for mouse-follow switching.

## Expected behavior

- **Local snake** follows mouse target smoothly (no teleport).
- **AI snake** follows the user by default.
- On likely/confirmed artifact intent, AI switches to **fast target** mode.
- Artifact context is activated and shown in the tutorial/chat dock.
- Follow-up questions stay scoped to the active artifact context.

## Manual mouse-event validation

1. Open `ananta tui`.
2. Press `Ctrl+S`, then `o`.
3. Move mouse across NAV and CONTENT panes.
4. Hover an artifact row briefly or click it.
5. Confirm markers appear in the dock:
   - `[mouse-follow]`
   - `[artifact-intent]`
   - `[ai-fast-target]`
   - `[artifact-chat-active]`

If mouse events are unavailable, keep using keyboard steering and `:mouse` command toggles.

## Inline viewer and artifact chat

- Confirmed text artifacts open in the inline viewer path by default.
- Binary or missing files fall back to metadata-only context.
- External editor launch is not part of the mouse artifact explanation flow unless explicitly requested.

## Known caveats

- Terminal text selection can intercept mouse events; disable selection mode when testing.
- In `tmux`, enable mouse passthrough (`set -g mouse on`) if needed.
- SSH/remote layers may transform or drop mouse event sequences.

## Homepage E2E cast flow

The snake-mode cast includes mouse/artifact/chat proof markers and is generated by E2E recording:

```bash
PYTHONPATH=. .venv/bin/pytest -q tests/e2e/test_tui_mouse_snake_artifact_chat_cast.py -s
```

Live LM Studio variant:

```bash
ANANTA_E2E_LIVE_LMSTUDIO=1 \
ANANTA_TUI_LLM_API_BASE=http://127.0.0.1:1234/v1 \
ANANTA_TUI_LLM_MODEL=meta-llama_-_llama-3.2-1b-instruct \
PYTHONPATH=. .venv/bin/pytest -q tests/e2e/test_tui_snake_mode_live_lmstudio_e2e.py -s -rs
```
