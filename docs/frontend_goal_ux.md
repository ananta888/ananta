# Frontend: Goal-first UX and advanced panels

Advanced panels

- Surface policy, routing and verification details in an "Advanced" drawer or collapsible panel.
- Show a concise governance summary: policy checks passed/failed, routing reason, and whether local/hub fallback was used.

First-run guidance

- Provide a simple submit box for goals with an optional "Advanced" toggle.
- Show a short "first-run" hint explaining that the system will use safe defaults and how to access advanced controls.

Design note: keep the default experience minimal and surface advanced routing or verification details only on demand to avoid overwhelming new users.
