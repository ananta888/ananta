# Beta Feedback Plan

## Ziele
- Reale Nutzungsszenarien validieren
- Onboarding-Hürden identifizieren
- Sicherheits- und Permission-Model prüfen

## Kanäle
- Kurz-Survey nach 1 Woche Nutzung
- Strukturierte Interviews (30-45 Minuten)
- Issue-Triage im Repository

## Metriken
- Time-to-first-task
- Erfolgsrate bei Task-Ausführung
- Häufigkeit von Permission-Fehlern

## Ergebnis-Tracking
- Feedback in `docs/roadmap.md` priorisieren
- Maßnahmen in `todo.json` aufnehmen
