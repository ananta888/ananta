Ananta

Autonomous Networked Agents Navigate Trusted Artifacts.

Ananta ist eine offene, local-first Multi-Agenten-Plattform fuer sichere KI-gestuetzte Softwareentwicklung. Nicht einfach ein Chatbot mit Tools, sondern ein steuerbares Hub-Worker-System mit Policy, Rollen, Kontextkontrolle, Pruefung und nachvollziehbaren Artefakten.

Repository Schnellstart CodeCompass Angular-Ansicht
Goal -> Plan -> Task -> Verification -> Artifact Hub-kontrollierte Worker local-first / Docker optional Least-Privilege-Policies

Worum es konkret geht

Du beschreibst ein Ziel. Der Hub plant, priorisiert und delegiert Aufgaben. Worker laufen getrennt, arbeiten mit begrenztem Kontext und liefern Ergebnisse als pruefbare Artefakte zurueck. Dadurch bleiben Zugriff, Verantwortung und Nachvollziehbarkeit kontrollierbar.

Artefakt

Ergebnisse bleiben sichtbar, versionierbar und zitierbar.

Hub / Worker

Zentrale Orchestrierung, getrennte Laufzeiten, Docker-Stacks und kontrollierte Delegation statt Worker-zu-Worker-Wildwuchs.

CodeCompass

Graph-, Retrieval- und Artefakt-Schicht fuer gezielten Projektkontext statt alles an jedes Modell zu kippen.

Security by Design

Least Privilege, Rollen, Policies, Shell-Guardrails und kontrollierter Zugriff auf Code, Secrets und Infrastruktur.

CLI und UI

CLI-first moeglich, Angular-Frontend vorhanden, Operator-TUI fuer sichtbare Agentenarbeit und Demos.

Local-first

Ollama, LM Studio, OpenAI-kompatible APIs und Docker-Setups sind als umschaltbare Provider-/Runtime-Pfade gedacht.

Learning Layer

Das Ananta Strategy Game dient als anschauliche Lern- und Demo-Schicht fuer Agenten, Policies, Kryptographie und sichere KI.

Schnellstart

Der aktuelle Code-Stand dokumentiert mehrere Einstiege: CLI-only, lokaler Hub/Worker ohne Docker, Angular-Frontend, Lite-Docker-Stack und ein Quickstart-Image ohne Ollama. Der offizielle Startpunkt liegt im README des Repositories.
